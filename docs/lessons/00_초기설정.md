# 초기설정

초기설정 단계에서는 실제 기능 구현을 시작하기 전에 학습 순서와 프로젝트 기준을 정리하고, 학습 문서와 챗봇 실습을 함께 다룰 Streamlit 기본 UI를 구성합니다.

### 목표

- 단계별 학습 목표와 구현 순서 정리
- `uv sync`로 실행 환경을 준비하는 방법 정리
- `.gitignore` 설정
- `.env.example`에 필요한 환경 변수 샘플 준비
- 학습용 Markdown 문서를 둘 위치와 관리 방식 정리
- Streamlit UI를 홈, 설명, 챗봇 탭으로 구성
- 설명 탭에서 학습용 Markdown 문서 목록과 내용을 볼 수 있도록 준비
- 챗봇 탭에 이후 LangChain 실습을 붙일 기본 영역 준비

- **완료 기준** : 실행 환경과 문서 구조가 준비되어 있고, Streamlit에서 홈/설명/챗봇 탭을 통해 학습 문서를 확인하며 이후 챗봇 기능을 붙일 수 있는 상태입니다.

---
### 자주 묻는 질문
#### `uv`란 무엇인가

`uv`는 Python 프로젝트의 실행 환경과 패키지를 관리하는 도구입니다. 이 프로젝트에서는 가상환경을 만들고, 필요한 패키지를 설치하고, 명령어를 프로젝트 환경 안에서 실행할 때 사용합니다.

예를 들어 다음 명령은 Streamlit을 현재 프로젝트 환경에서 실행합니다.

```bash
uv run streamlit run frontend/app.py
```

---
#### `TOML`과 `pyproject.toml`

`TOML`은 설정 파일을 작성하기 위한 문법입니다. Python 프로젝트에서는 프로젝트 이름, Python 버전, 필요한 패키지 같은 정보를 `pyproject.toml`에 적어둡니다.

이 프로젝트의 `pyproject.toml`에는 FastAPI, Streamlit, LangChain, LangSmith 같은 의존성이 정리되어 있습니다.

---
#### `uv sync`의 역할

`uv sync`는 `pyproject.toml`과 `uv.lock`을 기준으로 프로젝트 실행 환경을 맞추는 명령입니다.

주요 역할은 다음과 같습니다.

- `.venv` 가상환경 생성
- 필요한 패키지 설치
- 이미 설치된 패키지 버전 동기화
- `uv.lock`에 기록된 버전 기준으로 환경 재현

처음 프로젝트를 받은 뒤에는 보통 다음 명령으로 환경을 준비합니다.

```bash
uv sync
```

---
#### 가상환경이란 무엇인가

가상환경은 프로젝트마다 Python 패키지를 따로 관리하기 위한 독립된 실행 공간입니다. 프로젝트마다 필요한 패키지와 버전이 다를 수 있기 때문에, 가상환경을 사용하면 다른 프로젝트의 패키지와 섞이지 않게 관리할 수 있습니다.

이 프로젝트에서는 `uv sync`를 실행하면 `.venv` 폴더가 만들어지고, 필요한 패키지가 그 안에 설치됩니다.

PowerShell에서는 다음 명령으로 가상환경을 활성화합니다.

```powershell
.\.venv\Scripts\Activate.ps1
```

Windows의 Bash 환경에서는 다음 명령을 사용할 수 있습니다.

```bash
source .venv/Scripts/activate
```

Linux나 macOS의 Bash 환경에서는 다음 명령을 사용합니다.

```bash
source .venv/bin/activate
```

다만 `uv run ...` 명령은 가상환경을 직접 활성화하지 않아도 프로젝트 환경 안에서 명령을 실행해줍니다.

---
#### `.gitignore`

`.gitignore`는 Git에 올리지 않을 파일과 폴더를 정하는 설정 파일입니다.

이 프로젝트에서는 다음과 같은 파일을 제외합니다.

- `.env`처럼 개인 비밀값이 들어갈 수 있는 파일
- `.venv/`처럼 로컬에서 다시 만들 수 있는 가상환경 폴더
- `__pycache__/`, `.pytest_cache/` 같은 실행 중 생성되는 캐시
- `.claude/`, `AGENTS.md`처럼 개인 작업 도구나 에이전트 설정에 가까운 파일

반대로 `.env.example`은 실제 비밀값이 아니라 샘플 값을 담는 파일이므로 Git에 포함합니다.

---
#### `.env`와 `.env.example`

`.env`는 로컬 실행에 필요한 환경 변수를 적는 파일입니다. API 키처럼 개인마다 다르고 외부에 공개되면 안 되는 값은 `.env`에 넣습니다.

예시는 다음과 같습니다.

```env
OPENAI_API_KEY=your_openai_api_key
LANGSMITH_API_KEY=your_langsmith_api_key
```

`.env.example`은 어떤 환경 변수가 필요한지 알려주는 샘플 파일입니다. 실제 키 대신 `your_openai_api_key` 같은 placeholder 값을 적어두고, 프로젝트를 처음 실행하는 사람이 `.env`를 만들 때 참고하도록 둡니다.
