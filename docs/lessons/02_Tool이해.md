# Tool 이해

Tool 이해 단계에서는 LLM이 혼자 답할 수 있는 것과 애플리케이션의 도움을 받아야 답할 수 있는 것을 구분합니다. 아직 Tool을 직접 만들지는 않고, 왜 Tool이 필요한지 예시를 통해 먼저 이해합니다.

### 목표

- Tool이 무엇인지 이해하기
- LLM이 기본적으로 외부 세계를 직접 조회하지 못한다는 점 이해하기
- 웹 ChatGPT와 직접 만든 API의 차이 구분하기
- "현재 시간을 알려줘" 예시로 Tool이 필요한 상황 확인하기
- Tool을 붙였을 때의 요청 처리 흐름을 미리 살펴보기
- 다음 단계에서 직접 만들 Tool의 역할 정리하기

- **완료 기준** : Tool이 모델의 지식을 늘리는 기능이 아니라, 애플리케이션이 외부 정보나 기능을 모델에게 연결해주는 방법이라는 점을 설명할 수 있고, 현재 시간처럼 실행 시점의 정보가 왜 Tool 없이 정확히 처리되기 어려운지 이해한 상태입니다.

---
### Tool이 필요한 이유

LLM은 입력받은 메시지를 바탕으로 자연어 응답을 생성할 수 있습니다. 하지만 기본 상태의 LLM은 현재 시간, 오늘의 날씨, 데이터베이스에 저장된 사용자 정보, 사내 문서 검색 결과처럼 실행 시점에 외부에서 가져와야 하는 정보를 직접 조회하지 못합니다.

예를 들어 사용자가 다음처럼 질문한다고 가정합니다.

```text
현재 시간을 알려줘.
```

웹 ChatGPT에서는 이 질문에 답할 수 있는 것처럼 보일 수 있습니다. 웹 서비스에는 현재 시간, 웹 검색, 코드 실행 같은 내부 기능이 연결되어 있을 수 있기 때문입니다.

하지만 이 프로젝트의 API는 아직 사용자의 메시지를 LLM에 그대로 전달하고 응답을 받는 구조입니다. 현재 시간을 조회하는 기능을 백엔드에 만들지 않았고, 그 기능을 모델이 호출할 수 있도록 연결하지도 않았습니다. 그래서 모델은 정확한 현재 시간을 확인한 뒤 답하는 것이 아니라, 문맥과 일반 지식에 기대어 답하려고 합니다.

이때 필요한 연결점이 Tool입니다.

---
### Tool이란 무엇인가

Tool은 모델이 필요할 때 사용할 수 있도록 애플리케이션이 제공하는 기능입니다.

대표적인 예시는 다음과 같습니다.

- 현재 시간 조회
- 날씨 조회
- 웹 검색
- 데이터베이스 검색
- 파일 읽기
- 계산 실행
- 외부 API 호출

중요한 점은 Tool이 모델 내부에 자동으로 들어있는 능력이 아니라는 것입니다. 개발자가 애플리케이션에 기능을 만들고, 모델이 그 기능을 사용할 수 있는 방식으로 연결해야 합니다.

---
### 구현 흐름

현재 단계의 메시지 전송 흐름은 다음과 같습니다.

```text
사용자 메시지
  -> FastAPI POST /chat/message
  -> LangChain ChatOpenAI
  -> LLM 응답
```

Tool을 연결하면 흐름은 다음처럼 확장됩니다.

```text
사용자 메시지
  -> FastAPI POST /chat/message
  -> LLM이 Tool 필요 여부 판단
  -> 애플리케이션의 Tool 실행
  -> Tool 결과를 LLM에 전달
  -> LLM 최종 응답
```

이번 단계에서는 두 번째 흐름을 구현하지 않습니다. 먼저 Tool이 없는 상태에서 어떤 한계가 생기는지 확인하고, 다음 단계에서 직접 Tool을 만들어 연결합니다.

---
### 직접 확인하기

`http/tool_message.http`에는 Tool 없이 현재 시간을 물어보는 요청 예시가 들어 있습니다.

백엔드를 먼저 실행합니다.

```bash
uv run uvicorn backend.main:app --reload --port 8000
```

그 다음 VS Code REST Client 같은 도구에서 `Ask Current Time Without Tool` 요청을 실행합니다.

응답을 볼 때는 다음을 확인합니다.

- 모델이 실제 현재 시간을 정확히 조회했는가
- 현재 시간을 알 수 없다고 설명하는가
- 그럴듯한 시간을 추측해서 말하는가
- Tool이 필요하다는 식으로 한계를 설명하는가

응답 형태는 모델과 프롬프트에 따라 달라질 수 있습니다. 핵심은 답변 문장이 자연스러운지가 아니라, 애플리케이션이 현재 시간 조회 기능을 제공하지 않았는데도 모델이 정확한 실행 시점 정보를 보장할 수 있는지 확인하는 것입니다.

---
### 자주 묻는 질문
#### 웹 ChatGPT는 되는데 API는 왜 안 되는가

웹 ChatGPT와 직접 만든 API는 같은 모델을 쓰더라도 실행 환경이 다를 수 있습니다.

웹 ChatGPT에는 현재 시간, 검색, 파일, 코드 실행 같은 기능이 서비스 차원에서 연결되어 있을 수 있습니다. 반면 직접 만든 API는 개발자가 연결한 기능만 사용할 수 있습니다. 이 프로젝트의 현재 단계에서는 메시지 전송 기능만 만들었기 때문에, 모델이 현재 시간을 확인할 방법이 없습니다.

---
#### Tool은 프롬프트와 무엇이 다른가

프롬프트는 모델에게 어떻게 답해야 하는지 알려주는 입력입니다. Tool은 애플리케이션이 실제 기능을 실행하고 그 결과를 모델에게 전달하는 방법입니다.

예를 들어 프롬프트에 "현재 시간을 정확히 알려줘"라고 써도, 현재 시간을 조회하는 기능이 없다면 모델은 정확한 값을 확인할 수 없습니다. 반대로 현재 시간 Tool이 있으면 애플리케이션이 실제 시간을 계산하거나 조회한 뒤 그 결과를 모델에게 전달할 수 있습니다.

---
#### Tool은 항상 필요한가

항상 필요한 것은 아닙니다. 일반 개념 설명, 글쓰기, 요약, 코드 예시 작성처럼 모델이 입력과 학습된 지식을 바탕으로 처리할 수 있는 작업은 Tool 없이도 충분히 가능합니다.

하지만 실행 시점의 정보, 개인화된 데이터, 외부 시스템의 상태, 정확한 계산이나 검색 결과가 필요하다면 Tool을 고려해야 합니다.

---
#### 다음 단계에서는 무엇을 만드는가

다음 단계인 Tool 만들기에서는 현재 시간처럼 애플리케이션이 직접 제공할 수 있는 기능을 Tool로 만들고, 모델이 그 결과를 활용해 답하도록 연결합니다. 이 과정을 통해 "모델이 아는 척 답하는 구조"에서 "필요한 기능을 호출해 확인한 뒤 답하는 구조"로 확장합니다.
